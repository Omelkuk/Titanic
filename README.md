# Titanic

Решение классической задачи Kaggle по предсказанию выживаемости пассажиров. В проекте проведено сравнение различных алгоритмов: от простых классификаторов до сложных ансамблей.

Ниже представлены метрики обученных моделей, отсортированные по Accuracy.

| Модель | ROC-AUC | Accuracy | F1-Score | Precision | Recall |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **Voting Ensemble** | 0.8863 | **0.8212** | **0.7746** | 0.8088 | 0.7432 |
| **Stacking Ensemble**| **0.8864** | 0.8101 | 0.7606 | 0.7941 | 0.7297 |
| Random Forest (RF) | 0.8844 | 0.7989 | 0.7465 | 0.7794 | 0.7162 |
| XGBoost (XGB) | 0.8754 | 0.7989 | 0.7534 | 0.7639 | 0.7432 |
| LightGBM (LGB) | 0.8766 | 0.7933 | 0.7448 | 0.7606 | 0.7297 |
| Decision Tree (DT) | 0.8409 | 0.7933 | 0.7299 | 0.7937 | 0.6757 |
| Neural Network (NN) | 0.8739 | 0.7821 | 0.7310 | 0.7465 | 0.7162 |
| Ridge Regression | 0.7744 | 0.7821 | 0.7347 | 0.7397 | 0.7297 |
| k-NN | 0.8568 | 0.7765 | 0.7222 | 0.7429 | 0.7027 |

 Основные выводы:
1. Ансамблирование: Лучший результат по Accuracy (0.8212) показал `Voting Classifier`.
2. Разделяющая способность: `Stacking` и `Voting` показали практически идентичный и самый высокий `ROC-AUC` (~0.886), что говорит о высоком качестве ранжирования выживших.
3. диночные модели: Среди классических алгоритмов лидируют `Random Forest` и `XGBoost`.

 Библиотеки
 Конфиг: yaml, omegaconf
 
 Анализ данных: Pandas, NumPy
 
 Визуализация: Matplotlib, plotly
 
 Машинное обучение: Scikit-learn, XGBoost, LightGBM, Optune
 
 Нейронные сети:  PyTorch 

 Структура 
 config/ — конфигурационные файлы (`config.yaml`) для управления гиперпараметрами и путями.
 data/ — исходные и обработанные данные.
 eda/ — дополнительные скрипты для исследовательского анализа.
 src/ — папка с исходным кодом (модули для обучения и инференса).
 eda_exploration.ipynb — основной Jupyter Notebook с визуализацией и анализом.
 preprocessing.py — скрипт для очистки данных и генерации признаков (Feature Engineering).
 main.py — основной скрипт для запуска обучения и оценки моделей.
 factory.py — реализация паттерна "фабрика" для создания объектов моделей.
 optuna_study.db — база данных с результатами оптимизации гиперпараметров через Optuna.
 experiments.log— логирование процесса обучения и результатов тесто
